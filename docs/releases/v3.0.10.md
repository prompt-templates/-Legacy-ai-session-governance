# v3.0.10 Release Notes

Date: 2026-05-08 (UTC)

## Headline

Worktree session startup now respects the skip-worktree convention. When you open Claude Code inside a `git worktree`, governance correctly resolves `dev/SESSION_HANDOFF.md` / `dev/SESSION_LOG.md` from the main repo path instead of either failing with file-not-found or silently creating empty fallback files inside the worktree. Release verification harness gets the same treatment: `bash docs/qa/run_checks.sh` is documented to run from the main repo path — worktree-path execution triggers spurious `H01` (`.legacy_last_run` cross-tree drift) and `R27-10` (skip-worktree files absent in worktree) failures that are by-design per skip-worktree convention, not real failures. Bonus: small zh-TW residual sweep at the v3.0.8 row aligning with the v3.0.9 Round 5 formal-written register policy.

## What you'll feel

- If you use git worktrees for parallel features or per-session isolation, AI now reads `dev/SESSION_HANDOFF.md` / `dev/SESSION_LOG.md` from the main repo when they're skip-worktree'd — no more file-not-found at session start, no more empty fallback files appearing inside the worktree.
- Release verification gives consistent results regardless of execution path. AI knows to cd to the main repo before running `bash docs/qa/run_checks.sh`, so spurious `H01` / `R27-10` failures from worktree-path execution don't happen.
- The skip-worktree convention is now documented in `§3c` instead of leaving operators to discover it via spurious harness failures and infer the cause themselves.
- README.zh-TW.md v3.0.8 row reads as formal-written register end-to-end (`入面` / `令` / `用戶` tokens swept to match the v3.0.9 Round 5 sweep policy).

## What changed

### 1. `§1` Worktree fallback (mandatory)

`AGENTS.md` §1 (and `INIT.md` FILE 1 mirror) gained a new mandatory paragraph after the existing "create a minimal version" rule:

> Inside a git worktree, `dev/SESSION_HANDOFF.md` and `dev/SESSION_LOG.md` may be filtered out of the worktree's working tree (skip-worktree convention used to keep these local-only state files out of git history). When a worktree session starts and these files are absent in the worktree path, the AI must read them from the main repo path before falling back to the "create a minimal version" rule. Detect the worktree case via `git rev-parse --git-common-dir` (returns the shared `.git` directory; its parent is the main repo path) or by inspecting the current path for `.claude/worktrees/`. The "create a minimal version" rule applies only when both worktree and main repo lack the files (genuinely missing, not skip-worktree filtered).

Root cause addressed: prior `§1` startup sequence assumed single-tree checkout; worktree case was undefined, so AI either errored out on the file reads or generated empty fallback files in the worktree (both wrong — the files exist in main repo, just skip-worktree-filtered out of the worktree checkout).

### 2. `§3c` Phase 1 step 2 Canonical execution locus

`§3c` Phase 1 Machine Verification step gained a new sub-bullet:

> Canonical execution locus: harness must run from the main repo path (resolve via `git rev-parse --git-common-dir` parent dir, or `git worktree list` first entry). Worktree-path execution triggers spurious harness failures (`H01` `.legacy_last_run` cross-tree drift, `R27-10` skip-worktree files absent in worktree) that are by-design per skip-worktree convention — not real failures. AI must explicitly cd to main repo path before invoking the harness during release verification.

Root cause addressed: harness wasn't documented to require main-repo-path execution. AI invoking harness from a worktree path would surface 2 spurious failures (`H01` from cross-tree mtime drift + `R27-10` from skip-worktree files genuinely absent in worktree checkout). Operators had to debug the cause themselves; governance now states it explicitly.

### 3. zh-TW residual sweep at v3.0.8 row

Aligned with the v3.0.9 Round 5 sweep policy. Open Priority closed.

## Migration

No breaking changes. The new rules describe AI behavior in worktree edge cases that were previously undefined; existing single-tree workflows are unaffected.

Existing v3.0.x users:

- Re-run `INIT.md` against the same project root.
- `AGENTS.md` / `INIT.md` get the new `§1` Worktree fallback paragraph and `§3c` Canonical execution locus sub-bullet automatically merged.
- Existing `dev/SESSION_HANDOFF.md` / `dev/SESSION_LOG.md` skip-worktree status is unaffected — the new rules document the existing convention, they do not change it.
- If you've never used a worktree on this project, the only effect of upgrading is the harness counter bump (350 → 356) and the LATEST_STABLE_TAG bump (v3.0.9 → v3.0.10).
