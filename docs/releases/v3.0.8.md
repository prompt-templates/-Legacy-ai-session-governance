# v3.0.8 Release Notes

Date: 2026-05-05 (UTC)

## Headline

Install flow UX gets sharper — Profile picker's 6 options no longer collide into one paragraph, and the wizard prompt is now an independent message after `Governance framework ready`, so it's obvious that answering the wizard is optional, not part of finishing setup. The wizard's first question evolves from "give me a 1-sentence project description" into a main question plus three optional supplements (point me to reference files, URLs, or known decisions if handy), and AI now actively reads whatever you provide before drafting. Every assumption in the draft is tagged so you can tell at a glance which came from your input versus which AI estimated.

## What you'll feel

- Profile Selection's 6 options now show as a clean list (one per line, equals-aligned), instead of running together as a single paragraph.
- After picking your profile, you'll see two clearly separated messages: first `✅ Governance framework ready` (setup is done), then `💡 Optional next step — build a starter PROJECT_MASTER_SPEC.md?` (separate prompt). No more confusion about whether the wizard reply is required to finish install.
- Wizard's first question is no longer a single cold ask — it's a main question (briefly describe the project) plus three optional supplements: 📁 reference files, 🔗 relevant URLs, 📌 known decisions or constraints. An explicit escape phrase tells you "answering only the main question is sufficient".
- AI actively reads what you point at — local file paths get read, URLs get fetched, lookup keywords get grep'd — before drafting. No more silent fabrication of references / external products / competitor details when AI lacks ground truth.
- Every assumption in the draft is tagged: `[from your input]` (derived from your files / URLs / sentences) or `[my inference]` (AI estimated, lower confidence). You can spot-check sources without guessing which line is fact and which is AI guesswork.
- 4 prior Chinese-mixed-into-English defects in INIT.md / AGENTS.md cleaned up (e.g., `PR永久保留...` / `fixes真實 effective`).

## What changed

### 1. Install flow UX — Profile Selection reflow + two-message split

- `INIT.md` POST-INSTALL Profile Selection 6 options reflowed from inline `▸` (which markdown render squashed into one paragraph) to markdown list, one option per line, equals-aligned for visual scan.
- Install flow now splits into **two separate messages** (mandated as such, never combined):
  - **Message 1** — `✅ Governance framework ready.` (always sent; declares setup complete, profile saved, governance files in place).
  - **Message 2** — `💡 Optional next step — build a starter dev/PROJECT_MASTER_SPEC.md?` (skipped if spec already exists; offers 3-option choice — A run wizard now / B defer / C never ask again).
- Quick Start reference card follows as a third utterance.
- Triggered by: first real-install dogfood feedback after v3.0.7 (user observed "setup complete" + wizard prompt squashed into same screen, confusing whether the wizard reply was required).

### 2. Wizard Step 1 — main question + 3 optional supplements + escape phrase

- `dev/wizards/playbook.md` Step 1 restructured from "1-sentence cold question" to:
  - **Main question** — short open-ended ask (project description, who it's for).
  - **Optional supplements** (3 bullets): 📁 reference files (PRD / prior spec / meeting notes / design docs) / 🔗 relevant URLs (product page / docs / competitors) / 📌 known decisions (tech choice / deadlines / must-haves / must-not-haves).
  - **Escape phrase** — explicit "answering only the main question is sufficient" so supplements never read as a mandatory 4-field form.
- The prompt is rendered in the user's chat language; example phrasing in playbook is illustrative only.

### 3. Source-grounding discipline — AI actively reads provided sources

- New paragraph added to `AGENTS.md §3.6` + `INIT.md` FILE 1 mirror.
- Before drafting (Step 2a in playbook), AI MUST process every reference signal the user provided in Step 1:
  - Local file path → Read tool.
  - URL → WebFetch (or equivalent).
  - Keyword / project name → grep across the project / search where applicable.
- Silent fabrication of references / third-party products / competitor details is **prohibited**. If a source can't be processed (file missing, URL unreachable), surface the failure before drafting; do not invent content to fill the gap.

### 4. Labeled assumption list — `[from your input]` vs `[my inference]`

- `dev/wizards/playbook.md` Step 2c now requires every numbered assumption to start with a source tag:
  - `[from your input]` — derived from user-provided seed / files / URLs / decisions.
  - `[my inference]` — AI-estimated where ground truth was absent.
- Format invariant: each item in the assumption list carries its source tag at the start. Buried implicit inferences are explicitly prohibited.
- Replaces the prior un-labeled list (which made it hard for users to distinguish "AI read this from my files" vs "AI made this up").

### 5. Vague-input fallback — escape hatch user-language-agnostic

- Prior version hardcoded Cantonese `揀 A/B/C 或者俾多少少 context` as the verbatim escape hatch in both `AGENTS.md` / `INIT.md` §3.6 and `dev/wizards/playbook.md`.
- Reframed: the escape hatch must let the user pick A/B/C OR share more context, rendered in the user's chat language. Example phrasing only ("Pick A/B/C, or share a bit more context"); not a verbatim mandate.
- Hard rule: the free-form "share more context" path must NEVER be stripped — without it, the prompt regresses to a forced-choice form-fill.

### 6. Cleanup — Chinese-mixed defects + hardcoded example + drift sync

- 4 prior Chinese-mixed-into-English defects fixed (`AGENTS.md:306/308` + `INIT.md:320/322`):
  - `PR永久保留於 GitHub 作歷史，不可刪除.` → `PRs are preserved permanently on GitHub as history; do not delete.`
  - `confirm fixes真實 effective vs merely theoretical` → `confirm fixes are actually effective vs merely theoretical`.
- Hardcoded tech-stack example in `playbook.md:73` (`React + Node + PostgreSQL 而非 native iOS / Java`) generalized to placeholder pattern (`<dimension> 推 <chosen-option-A> 而非 <alternative-option-B>`) to satisfy `AGENTS.md §3b` anti-hardcoding rule.
- `dev/CODEBASE_CONTEXT.md` synced to `AGENTS.md §4a` (session-log archive gate is shell-evaluated at closeout, not Python at runtime; `session_log_maintenance.py` script preserved as dev self-test utility only). Prior wording from 2026-04-19 predated the §4a rewrite.
- `dev/wizards/README.md` Paradigm overview trimmed to pointer-only — `playbook.md` is the single source of truth for operational details (Step 1 frame, Step 2a source-grounding, Step 2c labeled assumptions). The README no longer duplicates these details inline.
