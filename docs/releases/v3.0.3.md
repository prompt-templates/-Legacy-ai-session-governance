### 2. §11a Reply Behavior governance

- New section `## 11a) Reply Behavior (Mandatory)` placed between §11 Output Contract and §12 Multi-Agent Session ID Standard.
- 5 rules:
  1. **Judgement-first** — state judgments / recommendations / opinions directly. No disguised open questions ("what do you think?", "should we A or B?") to push decisions back to the user. Assume professional capability.
  2. **Choice format** — when next step requires user choice and FPFR-style execution plan does not apply, present at most 3 options with one recommendation backed by verifiable evidence (file state, dependency, risk, stated user goal). Each option must be a viable beneficial path; warning-only options must be marked with the specific risk.
  3. **Ambiguity handling** — list at most 3 reasonable hypotheses and proceed with the most likely; ask clarifying questions only when missing data would change the answer's shape, and limit to ≤3 questions per round. Once user signals direction (apply / go / continue / chosen option letter), execute without re-confirming.
  4. **Fact verification** — verifiable facts (dates, numbers, regulations, names, quotes, citations) must be confirmed before stating. Unconfirmed = `UNVERIFIED`; this is distinct from `NA` which is reserved for genuinely missing values or non-applicable cases.
  5. **Plain-language surface text** — no `§` codes, internal rule IDs, file IDs, or invented terminology (Capital_Snake_Case, ProperNounMatrix-style names) as sentence subjects in user-facing reply. Lead with everyday language; § references and internal IDs may appear as parenthetical or end-of-line citations for traceability.
- New marker line: `<!-- MANDATORY REPLY DISCIPLINE — apply to every AI response: §11a -->` joins existing markers.

## Industry standards alignment

§11a Reply Behavior aligns with established collaborative-AI patterns: judgement-first reduces decision push-back loops; ≤3 options is the established cognitive cap for choice presentation; UNVERIFIED-vs-NA distinction prevents fact-confidence drift documented in API-doc-review patterns (§0b uses similar `Doc-reviewed` / `Test-verified` markers). The cap of 3 in choice format / hypotheses / questions per round is derived from real-world AI collaboration retrospectives where higher counts produced choice paralysis and over-clarification cycles.
