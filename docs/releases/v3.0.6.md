# v3.0.6 Release Notes

Date: 2026-05-01 (UTC)

## Headline

Quality-of-life polish on top of v3.0.5: prettier session boot/closeout visual cues across 6 styles, simpler one-line "paste hint" replacing 3-line conditional text, and a much shorter README install/upgrade flow (9 steps → 5 steps with a "behind the scenes" callout). README also explains, for the first time, why pasting the OPENING MESSAGE manually beats relying on the short `Follow AGENTS.md` trigger. Plus a pre-existing harness exit-code bug (R27-10) that was hidden until SESSION_LOG hit the 400-line trigger.

## What you'll feel

- Session start and end now show prettier ASCII art — sunrise mountains, sailing ship, star chart at boot; sunset over water, moonlit night, planted flag at closeout. 6 fresh styles randomized.
- The closeout block's caption is now one short line — `(paste as your next session's first message)` — instead of a three-line conditional explanation about same-machine vs cross-tool fallbacks.
- README Install drops from 9 cluttered steps (mixing your actions with AI's internal behavior) down to 5 actions you actually take, with AI's preflight/dry-run/backup explained in a short "Behind the scenes" callout.
- README Upgrading follows the same 5-step shape with a Behind-the-scenes callout — and the optional safe-upgrade prompt is now clearly labeled as "paste before step 3 for extra protection".
- README's "Resume in another AI CLI" now answers the question users actually have: *why paste the OPENING MESSAGE block instead of relying on `Follow AGENTS.md`?* — short answer: ~70-85% reliability vs ~95%+, paste removes guesswork.
- Harness reliability fixed: a pre-existing bash exit-code bug (R27-10) that masked until SESSION_LOG.md crossed the 400-line trigger is now patched.

### 5. README Quick Operations 3) — added "Why paste manually?" explanation

The most common new-user question — *why don't I just say `Follow AGENTS.md`?* — now answered inline:

> Governance is designed so AI auto-reads your saved handoff from `SESSION_LOG.md` — that's the self-contained intent. In practice, the short trigger `Follow AGENTS.md` only fires the full startup sequence ~70-85% of the time (varies by AI tool, model, and platform). The OPENING MESSAGE block is a longer explicit prompt — its first two lines anchor the receiving AI to read all 4 governance files in order, pushing reliability to ~95%+ across Codex / Claude Code / CoWork / Gemini CLI / web AI. One extra paste removes the guesswork. When in doubt, paste.

### 6. Pre-existing harness bug R27-10 fixed

- **Bug:** `legacy_checks.sh` R27-10 used pattern `if cmd && tests; then rc=0; elif z; then rc=0; else rc=$?; fi`. When the cmd-chain failed, the elif test was evaluated, replacing `$?` before the else captured it. Result: `rc` was always 1 in fall-through, never the python script's actual exit code (0 = no trigger, 2 = trigger met).
- **Why hidden:** `python session_log_maintenance.py --check` returned 0 when SESSION_LOG.md was below 400 lines, so the if-branch always succeeded. The bug only fired when SESSION_LOG actually triggered archive (return 2).
- **Fix:** restructured to capture exit code immediately after running the command, wrapped in `set +e/-e` to prevent `set -euo pipefail` from aborting on intentional non-zero exits.
- **File:** `docs/qa/legacy_checks.sh`.
