# v3.0.7 Release Notes

Date: 2026-05-03 (UTC)

## Headline

New onboarding wizard system: AI now drafts your `PROJECT_MASTER_SPEC.md` or `RUNBOOK.md` from a single sentence describing your project. Replaces the prior 5-7 step structured Q&A schema, which proved too rigid for vague long-term project visions. Plus a root-fix to the matrix-qc audit tool (boundary-aware divergence rules + prescriptive-verb ban) so audits stay description-only and don't cry wolf on intentional install-template-vs-runtime differences.

## What you'll feel

- First-time users no longer face a blank `PROJECT_MASTER_SPEC.md` template — give AI a 1-sentence project description, AI drafts the full file with every assumption surfaced as a numbered list for you to spot-check by index or natural language.
- After AI drafts, you correct items inline ("③⑤ 改成 …" or free-form), AI re-drafts the whole doc + a refreshed assumption list. When you say "OK 寫" / "write it", AI writes directly without a redundant "我寫入嗎？" round-trip.
- Long-term project visions that don't fit a structured Q&A form (marketing campaigns, research projects, side-projects with vague scope) are now first-class supported — paradigm aligned with modern Agentic AI collaboration, not blank-template fill-in.
- Optional template fields (References / prior art / specific dates without a stated timeline) get `(待補)` / `TBD` instead of fabricated entries — no more hallucinated content slipping into your spec drafts.
- Landing page now describes the wizard system as a feature card so first-time visitors discover it without reading the README.
- Matrix-QC audit tool no longer flags intentional install-template-vs-live-registry divergence as drift — prior false-positive on "verification method column generic" boundary now correctly classified as boundary-aware.
- Audit tool findings stay description-only ("rows X and Y diverge") — never prescriptive ("needs sync" / "should match" / "must be updated"). Fix decisions belong to the human, not the audit tool.

### 2. Matrix-QC audit tool root-fix — boundary-aware divergence + prescriptive-verb ban

- Internal dev tool at `.claude/agents/matrix-qc.md` (122 → 150 lines). Local-only / gitignored — does not ship via INIT.md.
- New `## Boundary-Aware Divergence` section lists 3 known intentional same-rule-different-place pairs: (1) INIT.md FILE 6 install template vs `dev/DOC_SYNC_CHECKLIST.md` live registry; (2) INIT.md vs repo runtime files (R27-19/20 self-containment); (3) README × 4 vs internal scripts (R27-11..18 hide internal tooling). Phase B Dimensions 3 / 4 / 8 cross-reference this section before flagging FAIL on cross-file comparison.
- Hard rule #2 tightened with prescriptive-verb ban list. Phase B/C output may not use `needs sync`, `should match`, `must be updated`, `whole-file sync needed`, `should be migrated`, `must align`. Findings must use neutral comparative language: `rows X and Y diverge`, `value mismatch detected`, `wording differs`. Fix decisions belong to the human, not the audit.

### 3. Playbook dogfood patches — 3 discipline rules from AI dogfood findings

- Step 4 close-out signal classification split into (a) soft closure (signals satisfaction; AI proposes write, asks confirm) vs (b) explicit write request (imperative write verb; AI writes directly, no extra turn). Eliminates redundant round-trip after `OK 寫` / `寫` / `write it`.
- Step 2 hallucination guardrail: optional fields without ground truth in seed/context (References / prior art / external products / specific dates without stated timeline) MUST write `(待補)` / `TBD` — no fabricated entries.
- Step 2 explicit per-field assumption: every filled template field gets at least one explicit assumption stating the primary inference. Implicit inferences buried in vague entries (e.g., bare `④ 技術 stack`) are prohibited; assumption list must surface the actual choice.
