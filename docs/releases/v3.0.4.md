# v3.0.4 Release Notes

Date: 2026-05-01 (UTC)

## Headline

Three changes that make the template more obvious to use: (1) the closeout block now says "OPENING MESSAGE — paste at next session" instead of generic "COPY/PASTE", so it's clear where it goes; (2) AI prints which source it used to seed the session at startup, so you can tell whether your paste was needed or auto-fallback kicked in; (3) the README now shows the full daily lifecycle (start → work → close → resume) with a visual flow chart, instead of stopping at install + start.

## What you'll feel

- **You'll know what to do with the closeout block.** It's now labelled "NEXT SESSION OPENING MESSAGE" with a one-line hint underneath: "paste this as the first message of your next AI session". No more wondering where the block goes.
- **No more confusion about whether the AI is starting blank.** At startup the AI prints a single line — `Seed context: paste`, `Seed context: SESSION_LOG fallback`, or one of two combined forms — so you can tell exactly which source it used and whether your paste was needed.
- **The README now teaches the full daily flow.** A visual flow chart (in 4 languages) plus a 5-step walkthrough covers install → start → work → close → resume in one place, with an explicit panel about what happens if you forget to paste.
- **Release notes feel like release notes again.** Every release now follows a 5-section template that leads with "What you'll feel" — no more changelogs that read like internal commit logs. Existing v3.0.3 release notes were rewritten retroactively to match.
- **Forgot to paste? AI tells you what it did.** Same machine + same tool: AI auto-reads `dev/SESSION_LOG.md` fallback and prints `Seed context: SESSION_LOG fallback`. Different machine or web AI: AI starts blank — you see `Seed context: paste` is missing, paste was required.

## What changed

### 1. Closeout block heading rename

The closeout output's middle section was renamed `NEXT SESSION HANDOFF PROMPT (COPY/PASTE)` → `NEXT SESSION OPENING MESSAGE`, with a 3-line hint underneath:

```
NEXT SESSION OPENING MESSAGE
(paste this as the first message of your next AI session.
 Same machine + same repo + same tool? Optional — AI auto-reads SESSION_LOG fallback.
 Cross-tool / cross-machine / web LLM? Required.)
```

The internal SESSION_LOG section heading `### Next Session Handoff Prompt (Verbatim)` is unchanged — that's an internal reference for AI auto-fallback, not user-facing output.

Affected: `AGENTS.md` §4 (rule 3 + rule 6 skeleton), `INIT.md` mirror + install-time Quick Start "Resume in another AI tool" block, all 4 README variants' Quick Operations section.

### 2. Startup seed-context transparency

`AGENTS.md` §1 now mandates that immediately after the Boot Visual Cue, AI prints exactly one line stating which source seeded the session's startup context. Four forms:

- `Seed context: paste` — user pasted handoff at start of conversation
- `Seed context: SESSION_LOG fallback` — no paste; AI auto-read latest verbatim block from `dev/SESSION_LOG.md`
- `Seed context: paste + SESSION_LOG fallback (consistent)` — both present, content matches
- `Seed context: paste + SESSION_LOG fallback (diverged — used paste)` — both present, content differs; AI uses paste as primary per §2 rule 5 and surfaces the divergence

Without this line, the auto-fallback was invisible — users couldn't tell whether AI used their paste or fell back. Now every startup makes the source explicit.

### 3. README "How a session works" section

The previous "30-second Quick Start" (3 steps: install + paste + start) was replaced with "How a session works" (5 steps end-to-end: install one-time + start + work + close + next session paste), embedded with a localized SVG lifecycle flow chart and an explicit "forgot to paste in step 5?" panel.

Available in all 4 languages: `ref_doc/lifecycle_flow_en.svg` / `lifecycle_flow_tw.svg` / `lifecycle_flow_cn.svg` / `lifecycle_flow_ja.svg`. The Interactive Introduction callout was also strengthened from a one-line hint to an explicit "🆕 First time? — recommended before reading the rest of this README" prompt.

### 4. Release notes template

A new `docs/releases/_TEMPLATE.md` defines the 5-section format every release should follow: **Headline / What you'll feel / What changed / Validation / Upgrade path** (plus optional Tags/commits and Industry alignment). The mandatory **What you'll feel** section enforces a user-feel lead — sentence subjects must be the actual fact, not internal `§` codes or rule IDs as previously appeared in v3.0.3 notes. v3.0.3 release notes were rewritten retroactively to match the template.

## Upgrade path

Existing v3.0.x users:

- **Recommended:** re-run `INIT.md`. The two-stage gate (`INSTALL_ROOT_OK` + `INSTALL_WRITE_OK`) auto-backs up your current files. New closeout heading + seed-context transparency apply immediately to every subsequent AI session.
- **Or merge manually** if you have local customizations:
  - `AGENTS.md` §1: add the "Seed context transparency" block right after the Boot Visual Cue rule
  - `AGENTS.md` §4: rename rule 3's `NEXT SESSION HANDOFF PROMPT (COPY/PASTE)` → `NEXT SESSION OPENING MESSAGE` with the explanatory clause; update rule 6 skeleton with the 3-line hint
  - Mirror both into your `INIT.md`
- **Breaking changes:** none. Old SESSION_LOG entries with `### Next Session Handoff Prompt (Verbatim)` headings remain valid — that internal heading is unchanged, only the user-facing closeout output Section 2 was renamed.

## Industry standards alignment

The seed-context transparency rule reflects the principle that hidden auto-behavior is a primary source of user confusion in agentic-AI tools. Making the fallback explicit eliminates the "is the AI starting blank or did it pick something up?" guesswork. The 5-step lifecycle README structure follows the established pattern of leading with the loop a user repeats daily, not the one-time install — install gets a single step, the recurring 4 steps get equal weight.
